# 📌 Веб-застосунок «Дошка оголошень» (Announcement Board)

Сучасний, швидкий та функціональний веб-застосунок для публікації, пошуку та менеджменту приватних оголошень, розроблений за архітектурою **Server-Side Rendering (SSR)**. Проєкт демонструє практичне застосування технологічного стеку Node.js, роботу з реляційними базами даних через сучасні ORM-рішення, динамічну фільтрацію даних на стороні сервера та реалізацію безпечного асинхронного REST-взаємодії для CRUD-операцій.

---

## 🛠 Технологічний стек

- **Runtime Environment:** Node.js (v22+)
- **Backend Framework:** Express.js
- **Database ORM:** Prisma ORM (v5)
- **Database Engine:** SQLite (嵌入式 реляційна БД)
- **Template Engine:** EJS (Embedded JavaScript)
- **Styling:** Vanilla CSS3 (модульна та адаптивна сітка)

---

## 🚀 Функціональні можливості

### 1. Головна сторінка (`GET /`)

- **Карткова архітектура:** Візуально відокремлені блоки оголошень із відображенням категорії (динамічні емодзі-бейджі), заголовка, ціни у гривнях, форматованої дати та інтелектуального обмеження опису (до 100 символів із додаванням трикрапки `...`).
- **Повнотекстовий пошук:** Регістронезалежний (`case-insensitive`) пошук оголошень за підрядком у назві.
- **Гнучке сортування:** Сортування за часом створення через Prisma `orderBy` (опції «Новіші спочатку» та «Старіші спочатку»).
- **Розумна пагінація:** Посторінковий вивід строго по 10 оголошень на сторінку (параметри `skip` та `take`). Стан пошуку та сортування повністю зберігається у URL-параметрах при переходах між сторінками.

### 2. Сторінка перегляду оголошення (`GET /announcements/:id`)

- Повне відображення метаінформації без обрізання тексту.
- Інтеграція блоку контактних даних автора.
- **Безпечне видалення:** Асинхронний клієнтський скрипт, що викликає `confirm()` діалог для запобігання випадкових дій перед відправкою деструктивного запиту.

### 3. Форма створення оголошення (`GET & POST /announcements`)

- Інтерактивна форма з radio-кнопками для категорій (_Продаж, Послуги, Робота, Інше_).
- **Двоваріантна валідація:** Базові HTML5 атрибути на клієнті (`required`, `minlength`, `min`) дублюються суворою серверною валідацією.
- **Збереження стану (`Old Data Input`):** При помилках валідації форма рендериться повторно з виведенням локальних червоних повідомлень під конкретними невалідними полями, зберігаючи весь раніше введений текст та обрані radio-кнопки.

### 4. REST-видалення (`DELETE /announcements/:id`)

- Реалізовано через原生 JavaScript `fetch()` API з HTTP-методом `DELETE`.
- Сервер обробляє запит через `prisma.announcement.delete()` та повертає стандартний REST-статус `204 No Content`, після чого клієнт виконує миттєвий редірект.

## 📂 Структура проєкту

```text
announcement-board/
├── prisma/
│   ├── schema.prisma          # Схема даних та конфігурація моделей ORM
│   └── dev.db                 # Локальна база даних SQLite (генерується автоматично)
├── public/
│   └── styles.css             # Глобальні стилі застосунку
├── views/
│   ├── index.ejs              # Шаблон головної сторінки (список, фільтри, пагінація)
│   ├── announcement.ejs       # Шаблон детального перегляду та клієнтський JS
│   ├── new.ejs                # Шаблон форми створення з блоком помилок
│   ├── 404.ejs                # Сторінка обробки помилки 404 (Ресурс не знайдено)
│   └── error.ejs              # Сторінка обробки серверної помилки 500
├── .env                       # Конфігурація середовища та URL бази даних
├── app.js                     # Точка входу, архітектура маршрутів та Middleware
└── package.json               # Маніфест проєкту, скрипти та залежності
```

⚙️ Розгортання та запуск локально

1. Клонування проєкту
   Bash
   git clone <url-вашого-репозиторію>
   cd announcement-board
2. Встановлення пакетів
   Bash
   npm install
3. Налаштування оточення
   Створіть файл .env у кореневій директорії проєкту та додайте конфігурацію шляху до SQLite:

Фрагмент кода
DATABASE_URL="file:./dev.db" 4. Генерація Клієнта ORM та Міграція БД
Створення локальної бази даних та синхронізація схем:

Bash
npx prisma migrate dev --name init 5. Запуск застосунку
Режим розробки (з автоматичним перезапуском при зміні коду):

Bash
npm run dev
Стандартний Production-запуск:

Bash
npm start
Сервер буде розгорнуто за адресою: http://localhost:3000

🛡 Обробка помилок та безпека
Екранування даних: Використання синтаксису <%= %> в EJS гарантує автоматичне екранування будь-яких користувацьких вводів, повністю захищаючи застосунок від XSS-атак.

Централізований Error Handling: Усі асинхронні помилки маршрутів транслюються у спеціальні Express Middleware, запобігаючи падінню сервера та безпечно відображаючи кастомні сторінки 404 або 500 в залежності від типу збою.
